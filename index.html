Creating an object

There are two ways of constructing an object:

  • We can designate an object literal to a variable using curly braces. In other words, things that have curly brackets around them become object properties. 
    
    let bunny = {
      texture: "fluffy",
      "favourite food": "roated carrots",
      size: "small"
    };
  • Alternatively, we can create an object using a constructor function. In practice, this is rarely used. 
    
    let bunny = new Object();

Methods

Functions stored in properties are called methods. They are created in the same way as normal properties. ES6 allows you to omit the colon and the function keyword. 


     let bunny = {
       eat() {
         console.log("The bunny ate quickly.");
       }
     };

Object methods can be invoked by object.method(argument).

The this keyword

"This" is a keyword for functions. It is defined by how the function is invoked. Usually, it has nothing to do with how a function is created.

  • Inside a constructor function: "this" always refers to the newly created instance.
  • When an object's method is invoked, "this" refers to the object on the left of the dot. This means that if an object invokes a method it inherited from a prototype, this refers to the object that called it.
  • If a function is not invoked as a method, "this" is the global subject in non-strict mode, and undefined in strict mode.
  • Arrow functions do not have their own "this". They inherit it from the outer environment.

Property value shorthand

When creating an object literal, if the property value is the same as the value of a variable, we can omit the property value. 


    let rabbit = "cute";
    let bunny = "even cuter";

    let cuteAnimalCollection = {
      rabbit,
      bunny
    };
      

    /*{
      rabbit: "cute",
      bunny: "even cuter"
    }*/

Accessing properties

We can use the dot notation to access an object's property. The word after the dot must be the literal name of the property. Trying to access a property that does not exist returns undefined. 


     let bunny = {
       color: "white";
     };
     bunny.color; //Returns "white"

If the property name contains numbers, spaces, or special characters, we must use the bracket notation. 


      let bunny = {
        color: "white";
      };
      bunny["color"]; //Returns "white"

The expression inside the bracket will be evaluated. So we can also put a variable, or more complicated expressions, into the bracket.

Object-oriented programming

Object-oriented programming (OOP) follows a few key principles:

Encapsulation: This is achieved when each object keeps its state private, inside a class. Other objects don't have direct access to its state. Instead, they can only call a list of public functions - called methods.

Polymorphism: This is when code is written to use an interface automatically knows how to work with any number of different objects that provide that interface. In other words, polymorphic code work with any objects that support the interface it expects.

Inheritance: Writing new classes as subclasses of an existing class to inherit part of its behaviours.
